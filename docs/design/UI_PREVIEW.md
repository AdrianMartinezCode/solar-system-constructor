# Solar System Constructor - UI Preview

## Status

Draft

## Problem / context

This document collects UI sketches/previews for key panels and states, to help align on layout and interaction patterns before/alongside implementation.

## Goals / non-goals

- **Goals**:
  - Capture proposed UI layouts and states in a copy/paste-friendly format.
  - Serve as a visual reference while iterating on windowed UI and panels.
- **Non-goals**:
  - Precise pixel-perfect spec (this is intentionally lightweight/iterative).

## Key concepts / terminology

- **Panel**: a UI surface (often a window) that controls a feature (speed, generator, stats, editor).
- **State**: a mode the UI can enter (paused, hyperspeed, etc.).

(ASCII previews below)

## Data model / invariants

- Previewed panels assume a windowed UI shell (or an equivalent layout container).
- Values shown (time scale, presets, etc.) should map cleanly to underlying state/config.

## Key flows

- Adjust simulation speed via slider/presets and reflect mode changes (normal/paused/hyperspeed).

## Tradeoffs / alternatives

- ASCII previews are fast and diff-friendly, but less expressive than screenshots/mockups.

## Feature Integration Summary

### Simulation Speed Control

**Purpose**: Global time scale control for all orbital motion

**Key Features**:
- Range: 0x (pause) to 50x (hyperspeed)
- Real-time updates: Changes apply immediately to simulation
- Multiple input methods: Slider, text input, or preset buttons
- Visual feedback: Dynamic status label shows current speed tier
- Persistent state: Stored in Zustand, affects all orbiting bodies uniformly

**Technical Details**:
- Updates `timeScale` in global Zustand store
- Applied in `tick()` function: `time += delta * timeScale`
- All orbital calculations use the scaled time automatically
- No performance impact (simple multiplication)
